🚨 발생한 문제들

1. Content Security Policy (CSP) 차단

문제: "Refused to load the script" 오류 발생

원인: Next.js의 CSP 설정이 카카오맵 스크립트 로딩을 차단

// CSP 설정이 스크립트 로딩을 차단 Content-Security-Policy: script-src 'self' 'unsafe-inline'

2. LatLng 생성자 오류

문제: "TypeError: window.kakao.maps.LatLng is not a constructor"

원인: 카카오맵 API가 완전히 로드되기 전에 생성자에 접근

// 잘못된 방식 - API 로드 전 접근 const marker = new window.kakao.maps.LatLng(lat, lng)

3. 비동기 스크립트 로딩 문제

문제: 동적 스크립트 생성 방식의 불안정성

원인: 스크립트 로딩 타이밍과 API 초기화 순서 문제

🔧 해결 과정

1단계

CSP 설정 제거

개발 환경에서 CSP 설정을 주석 처리하여 스크립트 로딩 차단 해제

// next.config.js // async headers() { // return [ // { // source: '/(.*)', // headers: [ // { // key: 'Content-Security-Policy', // value: "..." // }, // ], // }, // ] // },
2단계

Next.js Script 컴포넌트 사용

동적 스크립트 생성 대신 Next.js의 공식 Script 컴포넌트 활용

import Script from 'next/script' <Script src={`https://dapi.kakao.com/v2/maps/sdk.js?appkey=${apiKey}&autoload=false`} strategy="afterInteractive" onLoad={handleScriptLoad} onError={handleScriptError} />
3단계

안전한 API 초기화

kakao.maps.load() 콜백을 사용하여 API 완전 로드 후 초기화

const handleScriptLoad = () => { if (window.kakao && window.kakao.maps) { window.kakao.maps.load(() => { createMap() // API 완전 로드 후 지도 생성 }) } }
